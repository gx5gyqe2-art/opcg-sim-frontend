# OPCG SIM 機能分類（ユーザ目線）

OPCG SIM（ワンピースカードゲーム シミュレータ）の全機能を、**ユーザが何をしたいか**という
目線で分類した資料です。メインメニュー（`src/ui/GameStart.tsx`）の構成
（「Deck & Cards」「Simulation」）に沿って整理しています。

---

## 全体像

```
遊ぶ           ─ 1人回し / オンライン対戦 / VS CPU
準備する       ─ デッキ作成・一覧 / カードリスト / 画像DL
対局中の操作   ─ 盤面操作 / 戦闘 / 効果処理 / 情報確認
（補助）       ─ 自動復元 / レスポンシブ / オフライン / ルール強制
```

内部的には「1人回し＝サンドボックス（自由操作）」と「対戦／自動＝正規ルールエンジン処理」の
2系統に分かれます。

---

## 1. 遊ぶ（対戦・シミュレーション）

| 機能 | メニュー表記 | 内容 | ユーザの目的 | 主な実装 |
|---|---|---|---|---|
| 1人回しモード | 1人回しモード（Solo Sandbox） | 両プレイヤーを自分で操作する自由盤面。ルール強制なし | デッキの試運転・ムーブ確認・練習 | `screens/SandboxGame.tsx` |
| オンライン対戦 | 対戦モード（Online Multiplayer） | ルーム作成/参加で通信対戦（WebSocket）。P1/P2/観戦の役割選択 | 友人との対人戦 | `screens/RoomLobby.tsx`, `screens/SandboxGame.tsx`, `/ws/sandbox/{game_id}` |
| VS CPU（自動） | 自動モード（VS CPU・Rule Enforced） | ルール強制エンジン付きのCPU対戦 | 正規ルールでの対戦・効果の自動処理確認 | `screens/RealGame.tsx`, `/api/game/*` |

---

## 2. デッキとカードを準備する

| 機能 | メニュー表記 | 内容 | ユーザの目的 | 主な実装 |
|---|---|---|---|---|
| デッキ作成 / 一覧 | Deck Builder | デッキの新規作成・編集・保存・削除・一覧 | 自分のデッキを組む・管理する | `screens/DeckBuilder.tsx`, `/api/deck*` |
| カードリスト（閲覧専用） | Card Catalog | 全カードをカタログ表示（編集不可の閲覧モード） | カードを探す・効果テキストを確認 | `screens/DeckBuilder.tsx`（viewOnly） |
| 画像一括ダウンロード | 📥 画像一括DL | 全カード画像を事前キャッシュ | オフライン/低速回線でも快適に遊ぶ | `utils/imageAssets.ts`, `/api/cards` |

---

## 3. 対局中（ゲーム盤面）でできる操作

ゲーム中、ユーザがカード／盤面に対して行える操作群です。

- **盤面ゾーン操作**：手札・場（キャラ/リーダー/ステージ）・デッキ・トラッシュ・ライフ・ドン!!・コストエリア
- **カードアクション**（`ui/CardActionMenu.tsx`）：プレイ/移動、レスト⇄アクティブ、効果付与、KO など
- **戦闘フロー**：アタック宣言 → ブロック → カウンター → ダメージ（フェーズ：DON/メイン/バトル/エンド）
- **カード効果の処理**：効果の発動、対象選択、任意確認（使う/使わない）、トリガー解決
- **情報確認**：
  - カード詳細表示（`ui/CardDetailSheet.tsx`）
  - 盤面ズーム/インスペクト（`ui/InspectOverlay.tsx`）
  - アクションログ（`ui/ActionLog.tsx`）
  - 効果トースト通知（`ui/EffectToast.tsx`）

---

## 4. 利便性・縁の下の機能（ユーザが意識しにくいが恩恵を受ける）

- **進行状態の自動復元**：リロード/クラッシュしてもセッション復帰（`sessionStorage`）
- **レスポンシブ対応**：モバイル/PCで画面を自動スケーリング（`GameStart` の content scale 等）
- **オフライン対応**：画像キャッシュによる低通信運用
- **ルール自動強制エンジン**（自動/対戦モード）：カード効果テキストを解析し、トリガー・継続効果・
  除去保護・置換効果まで自動裁定（バックエンド `opcg_sim/src/core/effects/`）
